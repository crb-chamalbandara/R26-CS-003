Extension Analyzer
Malicious browser extension detection via ML + sandbox execution
In Development
Research Question
Can a combination of ML-based static analysis and sandboxed dynamic execution accurately detect malicious browser extensions before they cause harm?
Static ML Sandbox Execution JS AST Analysis Behaviour Logging
Architecture Pipeline
Extension Upload
      |
      v
+-------------+    manifest.json + JS source
| Static      |----------------------------->  ML classifier
| Analysis    |    permissions, API patterns   (score_static)
+-------------+
      |
      | (if score_static > threshold)
      v
+-------------+    Playwright headless Chromium
| Sandbox     |----------------------------->  Behaviour logger
| Execution   |    network, DOM, cookie ops    (score_dynamic)
+-------------+
      |
      v
  Fused verdict (weighted average)
Tech Stack
Static: scikit-learn / XGBoost on manifest.json + JS AST features (esprima/acorn)
Sandbox: Playwright Chromium with --load-extension
Network capture: Playwright route interception / mitmproxy
DOM monitoring: MutationObserver injected via CDP

Detection Engine Reference
Model pipeline & MITRE ATT&CK coverage
Isolation Forest (IF)
200 trees · 4 IAT timing features · unsupervised anomaly detection on network timing patterns
RF Classifier
10 HTTP behavior features · AUC 0.99 · supervised classification trained on real web-bot dataset
Heuristic Rules
5 rule checks: regular timing, foreground idle, background tab, extension origin, same-endpoint repeat
MITRE ATT&CK
T1071.001 Web Protocol   T1571 Non-Std Port   T1041 C2 Exfil
Risk Fusion
IF×0.40 + RF×0.20 + Rep×0.25 + Heuristic×0.15 — adaptive weights, BEACON threshold ≥ 0.60
Auto-Block
Hosts scoring ≥ 0.80 are automatically blocked via Playwright route interception. Unblock from Blocked Hosts above.
