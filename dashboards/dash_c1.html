C3
C2 Beacon Detector
Browser execution-aware C2 beaconing detection via CDP + ML
In Development
Research Question
Can runtime context-aware network monitoring inside the browser detect C2 beaconing behaviour from malicious extensions, even when traffic uses legitimate platforms such as Google Docs, Discord, or GitHub?
CDP Interception Isolation Forest LSTM Timing Context Tagging
Architecture Pipeline
Browser Network Events (CDP / Playwright route)
      |
      +-- Tab ID, Extension ID, User-active flag
      +-- Request URL, method, headers, payload size
      +-- Timing (inter-request intervals)
              |
              v
        Feature vector per request sequence
              |
              v
      ML Beaconing Classifier
      (periodic timing, encoded payloads, covert channels)
              |
              v
    Block + Flag -> Extension quarantined
Tech Stack
Interception: Playwright route() or Chrome DevTools Protocol (CDP)
Context tagging: CDP Network.enable + Target.getTargetInfo
ML model: Isolation Forest / LSTM on request timing sequences
Storage: SQLite (in-process, zero-config)
