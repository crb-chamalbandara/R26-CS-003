C1
Extension Analyzer
Malicious browser extension detection via ML + sandbox execution
In Development
Research Question
Can a combination of ML-based static analysis and sandboxed dynamic execution accurately detect malicious browser extensions before they cause harm?
Static ML Sandbox Execution JS AST Analysis Behaviour Logging
Architecture Pipeline
Extension Upload
      |
      v
+-------------+    manifest.json + JS source
| Static      |----------------------------->  ML classifier
| Analysis    |    permissions, API patterns   (score_static)
+-------------+
      |
      | (if score_static > threshold)
      v
+-------------+    Playwright headless Chromium
| Sandbox     |----------------------------->  Behaviour logger
| Execution   |    network, DOM, cookie ops    (score_dynamic)
+-------------+
      |
      v
  Fused verdict (weighted average)
Tech Stack
Static: scikit-learn / XGBoost on manifest.json + JS AST features (esprima/acorn)
Sandbox: Playwright Chromium with --load-extension
Network capture: Playwright route interception / mitmproxy
DOM monitoring: MutationObserver injected via CDP

C3
C2 Beacon Detector
Browser execution-aware C2 beaconing detection via CDP + ML
In Development
Research Question
Can runtime context-aware network monitoring inside the browser detect C2 beaconing behaviour from malicious extensions, even when traffic uses legitimate platforms such as Google Docs, Discord, or GitHub?
CDP Interception Isolation Forest LSTM Timing Context Tagging
Architecture Pipeline
Browser Network Events (CDP / Playwright route)
      |
      +-- Tab ID, Extension ID, User-active flag
      +-- Request URL, method, headers, payload size
      +-- Timing (inter-request intervals)
              |
              v
        Feature vector per request sequence
              |
              v
      ML Beaconing Classifier
      (periodic timing, encoded payloads, covert channels)
              |
              v
    Block + Flag -> Extension quarantined
Tech Stack
Interception: Playwright route() or Chrome DevTools Protocol (CDP)
Context tagging: CDP Network.enable + Target.getTargetInfo
ML model: Isolation Forest / LSTM on request timing sequences
Storage: SQLite (in-process, zero-config)
