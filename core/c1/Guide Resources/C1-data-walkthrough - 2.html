Component 1 — Data Collection Walkthrough
How to prepare the datasets so development can start
This page turns the Data Collection phase into a step-by-step action plan, from downloading sources to building the final training CSV.

Goal of this phase

You need a clean, labeled dataset with both malicious and benign extensions, ready for ML training. The output of this phase is a single CSV with one row per extension and a label column (1 = malicious, 0 = benign).

You are ready to start development when you have: (1) all malicious sources downloaded, (2) a benign baseline list, (3) a unified CSV structure, and (4) a small test batch proving your pipeline works end-to-end.

Datasets you will use

Malicious

chrome-mal-ids

Blocklist of known malicious extension IDs. Best for hash list + ID-based manifest collection.

git clone https://github.com/mallorybowes/chrome-mal-ids.git
Malicious

palant/malicious-extensions-list

Curated list with behavior descriptions. Use it to design sandbox checks later.

git clone https://github.com/palant/malicious-extensions-list.git
Malicious

Chrome-Stats malware removals

Extensions removed by Google as malware. High-quality labels but may require archived manifests.

https://chrome-stats.com/search?obsoleteReason=malware
Malicious

refade/GoogleChromeExtension

990 malicious extensions with 1,098 features extracted. Fastest way to start ML training.

git clone https://github.com/refade/GoogleChromeExtension.git
Benign (self)

Top Chrome Web Store extensions

Collect 500–1000 popular, long-lived extensions using CRXcavator + Web Store metadata.

https://crxcavator.io/api/v1/report/EXTENSION_ID

Walkthrough — end-to-end steps

1
Create a local folder called datasets and clone the three GitHub sources (chrome-mal-ids, palant list, refade dataset). This gives you the initial malicious IDs and one ready-made feature CSV.
2
From each malicious list, extract the Extension IDs into a single file (one ID per line). Remove duplicates so you have a clean malicious ID pool.
3
Visit Chrome-Stats malware removals and export the ID list. Add these IDs to the same pool.
4
For each malicious ID, collect metadata from CRXcavator (permissions, manifest info). If CRXcavator has no data, skip or mark as “archived needed”.
5
Build a benign list: pick 500–1000 top extensions, then fetch their permissions and manifest metadata via CRXcavator. Label them as benign (0).
6
Create one unified CSV schema. Merge malicious and benign rows into one dataset. Ensure consistent columns (missing permissions become 0).
7
Run a quick validation: check label balance, ensure no empty rows, and confirm every row has an extension_id and label.

Ready-to-start checklist

All malicious sources cloned and IDs merged
Benign ID list created (500–1000)
CRXcavator API tested on 5 IDs
Unified CSV schema created
At least 50 rows collected end-to-end
Label balance checked
When you can collect 50 rows end-to-end without errors, you are ready to scale to 2000+ rows and start model training.
